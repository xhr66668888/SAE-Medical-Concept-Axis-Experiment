## 每次开始使用

进入项目并激活 venv：

```bash
cd /home/haoranxu/sae
source .venv/bin/activate
```

建议设置线程参数：

```bash
export OMP_NUM_THREADS=8
export MKL_NUM_THREADS=8
export TOKENIZERS_PARALLELISM=false
```

检查环境：

```bash
python scripts/check_hardware.py
```

## 开启 Intel XPU

只需要做一次：

```bash
sudo usermod -aG render,video $USER
reboot
```

重启后：

```bash
cd /home/haoranxu/sae
source .venv/bin/activate
id -nG
python scripts/check_hardware.py
```

如果 `id -nG` 里出现 `render` 和 `video`，并且 `check_hardware.py` 显示 `xpu available: True`，就可以用 Intel GPU 跑。

## Hugging Face 登录

Gemma 模型一般需要先在 Hugging Face 网页接受 license，然后登录：


```bash

```

## 推荐运行顺序

### Step 1：MVP 冒烟测试

XPU 可用后，先跑默认的 Gemma 3 1B + Gemma Scope 2：

```bash
python scripts/step1_mvp.py --device xpu --dtype bfloat16
```

如果 XPU 还没修好，先用 CPU + 270M 跑通链路：

```bash
python scripts/step1_mvp.py \
  --preset gemma3-270m-it-res9 \
  --device cpu \
  --dtype float32
```

### Step 2：生成 prompts

默认数据已经存在于：

```text
data/diabetes_contrastive_prompts.csv
```

重新生成：

```bash
python scripts/step2_generate_prompts.py \
  --icd-csv data/icd_diagnosis_ccs.csv \
  --atc-csv data/atc_drug_hierarchy.csv \
  --output data/diabetes_contrastive_prompts.csv
```

### Step 3：概念轴实验

XPU 路线：

```bash
python scripts/step3_concept_axis.py \
  --prompts data/diabetes_contrastive_prompts.csv \
  --device xpu \
  --dtype bfloat16 \
  --output-dir outputs/axis
```

小规模冒烟：

```bash
python scripts/step3_concept_axis.py \
  --prompts data/diabetes_contrastive_prompts.csv \
  --max-prompts-per-class 2 \
  --device xpu \
  --dtype bfloat16 \
  --output-dir outputs/axis_smoke
```

CPU fallback：

```bash
python scripts/step3_concept_axis.py \
  --preset gemma3-270m-it-res \
  --prompts data/diabetes_contrastive_prompts.csv \
  --max-prompts-per-class 1 \
  --device cpu \
  --dtype float32 \
  --output-dir outputs/axis_cpu_smoke
```

### Step 4：跨层 SAE 特征路线

XPU 路线：

```bash
python scripts/step4_trace_circuit.py \
  --prompts data/diabetes_contrastive_prompts.csv \
  --device xpu \
  --dtype bfloat16 \
  --output-dir outputs/circuit
```

本机建议先跑小规模：

```bash
python scripts/step4_trace_circuit.py \
  --prompts data/diabetes_contrastive_prompts.csv \
  --layers 13 \
  --max-prompts-per-group 1 \
  --top-k 3 \
  --device xpu \
  --dtype bfloat16 \
  --output-dir outputs/circuit_smoke
```

CPU fallback：

```bash
python scripts/step4_trace_circuit.py \
  --preset gemma3-270m-it-res \
  --prompts data/diabetes_contrastive_prompts.csv \
  --layers 9 \
  --max-prompts-per-group 1 \
  --top-k 3 \
  --device cpu \
  --dtype float32 \
  --output-dir outputs/circuit_cpu_smoke
```

## 常见问题

### `xpu available: False`

优先检查用户组：

```bash
id -nG
ls -l /dev/dri
```

修复：

```bash
sudo usermod -aG render,video $USER
reboot
```

### 缺 Hugging Face 权限

现象通常是模型下载失败或 401/403：

```bash
huggingface-cli login
```

并确认你已在 Hugging Face 接受 Gemma 模型 license。

### OOM 或内存不足

先缩小实验：

```bash
--max-prompts-per-class 1
--max-prompts-per-group 1
--layers 13
--top-k 3
```

或者改用：

```bash
--preset gemma3-270m-it-res
--device cpu
--dtype float32
```

### 维度不匹配

加 `--debug`：

```bash
python scripts/step3_concept_axis.py --debug
python scripts/step4_trace_circuit.py --debug
```

Step 1 会默认打印 token shape、target token、residual shape 和 SAE activation shape；如果要看更多 feature，可以用：

```bash
python scripts/step1_mvp.py --top-k 10
```

## 重新安装 venv

如果环境坏了，可以删除并重建：

```bash
cd /home/haoranxu/sae
rm -rf .venv
export UV_CACHE_DIR=/home/haoranxu/sae/.uv-cache
export UV_PYTHON_INSTALL_DIR=/home/haoranxu/sae/.uv-python
.tools/uv/uv venv --python 3.11 .venv
.tools/uv/uv pip install --python .venv/bin/python torch torchvision torchaudio --index-url https://download.pytorch.org/whl/xpu
.tools/uv/uv pip install --python .venv/bin/python -r requirements-intel.txt
```

然后：

```bash
source .venv/bin/activate
python scripts/check_hardware.py
```
